# AI-SDLC Matrix — Руководство по использованию

## 1. Установка зависимостей

> **Требование к Node.js:** версия `18.x` (точнее `>=18.17.1 <19.0.0`). Другие версии вызовут ошибку при сборке. Для переключения версий используйте [nvm](https://github.com/nvm-sh/nvm) или [nvm-windows](https://github.com/coreybutler/nvm-windows).

```
npm install
```

Команда устанавливает все зависимости из `package.json` в локальную папку `node_modules`.

---

## 2. Настройка окружения

Скопируйте `.env.template` в `.env` и заполните значениями вашего SharePoint-сайта:

| Переменная | Пример значения |
|---|---|
| `SP_SITE_URL` | `https://contoso.sharepoint.com/sites/AI-SDLCRoleMatrix` |
| `SP_FOLDER_ROOT` | `/sites/AI-SDLCRoleMatrix/SitePages/ai-sdlc-matrix` |
| `SP_WORKBENCH_URL` | `https://contoso.sharepoint.com/sites/AI-SDLCRoleMatrix/_layouts/workbench.aspx` |

Файл `.env` добавлен в `.gitignore` и никогда не попадает в репозиторий.

---

## 3. Сборка

```
gulp bundle --ship
gulp package-solution --ship
```

Выполните обе команды последовательно. Флаг `--ship` собирает минифицированный production-пакет.

**Результат сборки:** `sharepoint/solution/ai-sdlc-matrix.sppkg`

Этот `.sppkg`-файл — готовый к деплою пакет SharePoint. Внутри него собраны оба веб-парта.

---

## 4. Загрузка в SharePoint

1. Откройте сайт **App Catalog** вашего тенанта.
   - Если его нет — обратитесь к администратору тенанта или перейдите в SharePoint admin center → **More features → Apps → App Catalog**.
2. Перейдите в раздел **Apps for SharePoint** (левая навигация) → нажмите **Upload** или перетащите файл `ai-sdlc-matrix.sppkg`.
3. В появившемся диалоге нажмите **Deploy**.
   - Включите **«Make this solution available to all sites in the organization»**, если нужен деплой на все сайты тенанта без установки вручную на каждом.

---

## 5. Добавление приложения на сайт

После загрузки в App Catalog:

1. Перейдите на нужный SharePoint-сайт.
2. Откройте **Site contents** → **New → App**.
3. Найдите **ai-sdlc-matrix** и нажмите **Add**.

После установки в редакторе страниц становятся доступны два веб-парта:

- **AI-SDLC Role Matrix**
- **MD Card**

---

## 6. Веб-парты

### AI-SDLC Role Matrix

Интерактивная матрица, показывающая вовлечённость каждой роли в фазы AI-SDLC на разных уровнях зрелости.

**Как это работает:**

Матрица построена по двум осям:
- **Колонки — Роли:** читаются автоматически из иерархии папок в библиотеке Site Pages (см. [Структура контента в SharePoint](#7-структура-контента-в-sharepoint) ниже).
- **Строки — Фазы:** фиксированный набор из 8 фаз:
  - Base AI Maturity (базовый уровень зрелости)
  - Planning (Планирование)
  - Requirements (Требования)
  - Design / Architecture (Дизайн / Архитектура)
  - Development (Разработка)
  - Testing (Тестирование)
  - Deployment / Release (Деплой / Релиз)
  - Maintenance (Поддержка)

В шапке — три кнопки переключения **уровня зрелости**:

| Уровень | Описание |
|---|---|
| AI Enabled | Команды используют AI-инструменты в рамках существующих процессов |
| AI-First | AI — основной рабочий режим |
| AI Native | Полностью AI-ориентированная модель поставки |

**Чтение матрицы:**

В каждой ячейке отображается цветная точка, показывающая степень вовлечённости роли в данную фазу на выбранном уровне зрелости:

| Символ | Значение |
|---|---|
| Большая фиолетовая точка | **Lead** — владеет фазой и ведёт её |
| Средняя бирюзовая точка | **Active** — основной участник |
| Маленькая янтарная точка | **Review** — ревьюит и согласовывает результаты |
| Маленькая серая точка | **On-demand** — консультируется по запросу |
| · (точка) | Не вовлечён |

Строка **Base AI Maturity** — особая: вместо точки вовлечённости в ней показывается иконка документа, если страница для этой роли существует. Клик открывает обзорную страницу уровня зрелости для роли.

**Клик по ячейке** открывает модальное окно с соответствующей страницей SharePoint — с полным описанием роли в данной фазе и на данном уровне. В модальном окне есть кнопка открыть страницу в новой вкладке.

**Роль Developer** — особый случай: в заголовке колонки появляется выпадающий список для переключения между под-специализациями (например General, Frontend, Backend). Матрица обновляется соответственно.

### MD Card

Загружает Markdown-файл (`.md`), хранящийся в SharePoint, и отображает его как отформатированную карточку на странице.

**Как это работает:**

1. Добавьте веб-парт **MD Card** на любую страницу SharePoint.
2. Откройте панель свойств (нажмите на иконку карандаша/редактирования на веб-парте).
3. Вставьте URL или серверно-относительный путь к `.md`-файлу в SharePoint.
   - Полный URL: `https://contoso.sharepoint.com/sites/MySite/Shared Documents/readme.md`
   - Серверно-относительный путь: `/sites/MySite/Shared Documents/readme.md`
   - Шаринговые ссылки SharePoint (вида `/:t:/s/...`) — тоже поддерживаются
4. Карточка отображает Markdown: заголовки, жирный текст, списки, блоки кода и т.д.

Если файл не найден — карточка покажет понятное сообщение об ошибке с указанием настроенного пути.

## 7. Структура контента в SharePoint

Матрица читает все данные из иерархии папок и страниц в SharePoint. Никаких внешних баз данных не используется.

### Структура папок

Весь контент находится внутри `SitePages/<корневая-папка-матрицы>/`:

```
SitePages/
└── ai-sdlc-matrix/
    ├── <Роль>/                        ← одна папка на роль (например Developer, PM, Architect)
    │   ├── Base-AI-Maturity/
    │   │   ├── ai-enabled.aspx
    │   │   ├── ai-first.aspx
    │   │   └── ai-native.aspx
    │   ├── Planning/
    │   │   ├── ai-enabled.aspx
    │   │   ├── ai-first.aspx
    │   │   └── ai-native.aspx
    │   ├── Requirements/
    │   ├── Design-Architecture/
    │   ├── Development/
    │   ├── Testing/
    │   ├── Deployment-Release/
    │   └── Maintenance/
    │
    └── Developer/                     ← Developer имеет дополнительный уровень вложенности
        ├── General/
        │   ├── Planning/
        │   │   ├── ai-enabled.aspx
        │   │   └── ...
        │   └── ...
        └── Frontend/
            └── ...
```

- **Папки ролей** — любая папка, созданная здесь, автоматически становится колонкой в матрице. Матрица обнаруживает их самостоятельно.
- **Папки фаз** — названия должны точно совпадать с перечисленными выше (с учётом регистра).
- **Страницы уровней** — должны называться `ai-enabled.aspx`, `ai-first.aspx` или `ai-native.aspx`.

### Столбец `Involvement`

Каждая `.aspx`-страница в иерархии должна иметь пользовательский столбец `Involvement` в библиотеке **Site Pages**. Это значение определяет точку, отображаемую в ячейке матрицы.

Допустимые значения (регистр не важен):

| Значение | Отображение в матрице |
|---|---|
| `lead` | Большая фиолетовая точка |
| `active` | Средняя бирюзовая точка |
| `review` | Маленькая янтарная точка |
| `on-demand` | Маленькая серая точка |
| *(пусто)* | Нет точки (не вовлечён) |

Чтобы задать значение: откройте страницу → нажмите **Page details** (или панель ⓘ) → найдите поле `Involvement` → выберите нужное значение.

> Если столбец `Involvement` отсутствует в библиотеке Site Pages, добавьте его вручную: **библиотека Site Pages → Add column → Choice** — назовите столбец exactly `Involvement` и добавьте пять значений выше.
